## 구현 판정

- 화면에 표시되는 중첩 인라인 본문과 지원 서식은 모델 변환에서 더 이상 조용히 누락되지 않는다.
- 표·목록·Section은 현재 DOCX 모델이 표현할 수 없으므로 저장 전에 명시적으로 차단한다.
- 비지원 블록이 있을 때 파일 쓰기와 안전 저장을 시작하지 않으므로 기존 원본은 변경되지 않는다.
- P01~P03 범위의 알려진 데이터 유실 회귀는 발견되지 않았다.

## 남은 위험

1. 브라우저, Word 등 외부 프로그램별 실제 클립보드 데이터 형식은 수동 붙여넣기 시험이 더 필요하다.
2. 비지원 블록은 텍스트 평탄화가 아니라 저장 차단 방식이므로 사용자가 일반 텍스트로 다시 붙여넣어야 한다.
3. Microsoft Word와 LibreOffice의 실제 DOCX 호환성은 V100-P04에서 검증해야 한다.
4. DOC COM 왕복과 `WINWORD.EXE`/임시 파일 정리는 V100-P05에서 검증해야 한다.
5. 확장자, 새 문서 상태, 도구 모음 동기화, 오류 분류는 V100-P06~P09로 남아 있다.

## 현재 판정

V100-P01~V100-P03은 개발 완료다. v1.0.0 전체 배포 승인은 P04~P10과 실제 오피스 환경 검증 전까지 보류한다.
