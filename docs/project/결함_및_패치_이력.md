# 결함 및 패치 이력

이 문서는 테스트에서 발견된 결함과 실제 수정 내용을 누적 관리한다. 개발자, 테스터, 기획자, Git 관리자는 작업 시작 전에 현재 버전의 미해결 항목을 확인한다.

## 관리 규칙

1. 결함은 `BUG-V{버전 숫자}-{일련번호}` 형식으로 등록한다.
2. 수정 작업은 버전 이력의 패치 ID와 연결한다.
3. 개발자는 원인, 변경 파일, 수정 내용, 자동 검증 결과를 기록한다.
4. 개발자 검증이 끝나도 테스터가 독립 재검증하기 전에는 `해결 확인`으로 표시하지 않는다.
5. 상태는 `접수`, `수정 중`, `재검증 대기`, `해결 확인`, `보류` 중 하나만 사용한다.
6. 패치를 수행할 때마다 새 결함 여부와 기존 회귀 테스트 결과를 이 문서에 누적한다.

## 상세 이력

### BUG-V100-001 — 비지원 블록 사용자 안내

- 발견: 독립 테스터의 V100-P02/P09 UI 검증
- 원인: `UnsupportedEditorContentException`이 일반 오류 분류의 `Unknown`으로 처리됨
- 수정: 저장 경로에서 전용 예외를 먼저 처리하고 Table/List/Section 종류, 저장 중단 이유, 일반 텍스트 재붙여넣기 해결 방법을 안내
- 검증: 콘솔 회귀 및 WPF UI 하네스에서 세 블록의 원인·해결 방법·저장 차단·원본 바이트 보존 확인
- 테스터 판정: 2026-07-12 해결 확인

### BUG-V100-002 — 글자 크기 표시 오차

- 발견: 독립 테스터의 V100-P08 단일 선택 UI 검증
- 재현: 16pt 텍스트의 문단 전체를 선택하면 도구 모음에 `15.75` 표시
- 원인: WPF가 16pt의 DIP 값(21.333...)을 21 DIP로 양자화하고, UI 상태 판독기가 이를 그대로 역환산함
- 수정 파일: `SimpleWordEditor/EditorFormatState.cs`, `SimpleWordEditor.Tests/Program.cs`
- 수정: 역환산 포인트가 정수와 0.25pt 이내이면 해당 정수로 표시하고, 그 밖의 값은 소수점 둘째 자리까지 유지
- 개발자 검증:
  - Debug 빌드: 경고 0, 오류 0
  - 콘솔 자동 회귀: 39건 통과, 0건 실패
  - WPF UI 하네스: `ToolbarSingle ... Size=16`, 혼합 선택 크기 빈 값 확인
  - 기존 비지원 블록 저장 차단과 원본 보존 시나리오 재통과
- 독립 테스터 재검증:
  - Debug/Release 빌드 경고 0, 오류 0
  - Debug/Release 자동 회귀 테스트 각각 39건 통과, 실패 0
  - 실제 Release WPF 화면에서 16pt 단일 선택이 `16`으로 표시됨
  - 혼합 선택에서 크기 입력란이 빈 값으로 표시되고 굵게·기울임·밑줄·정렬이 3상태로 유지됨
  - 8pt·72pt 저장/재열기 경계값 회귀 통과
  - 기존 Table·List·Section 저장 차단, 원본 보존 및 핵심 저장/확인 흐름 재통과
- 현재 상태: 해결 확인 (2026-07-12 독립 테스터)

## 다음 인수인계

테스터는 BUG-V100-002 재현 절차를 동일하게 실행해 16pt 표시를 확인하고, 8pt·72pt 및 혼합 선택도 회귀 확인한다. 통과하면 상태를 `해결 확인`으로 변경한다. 실패하면 실제 표시값과 DPI/Windows 배율을 함께 기록해 개발자에게 반환한다.
