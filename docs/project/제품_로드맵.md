# 간단 워드 편집기 제품 로드맵

## 문서 목적

이 문서는 v1.0.0 이후 어떤 버전에서 어떤 기능을 개발할지 정한다. 상세 구현은 각 버전의 `docs/plans/개발_기획서_vX.Y.Z.md`에서 다룬다.

로드맵은 개발 순서를 뜻한다. Word를 구할 수 없어 외부 호환성 검증이 장기간 보류되는 경우, 사용자가 승인하면 해당 검증을 배포 전 관문으로 남긴 채 다음 기능 개발을 진행할 수 있다. 단, 미검증 상태의 버전을 정식 배포해서는 안 된다.

## 우선순위 원칙

1. 데이터 유실 방지가 기능 수보다 우선한다.
2. 새 기능은 DOCX 저장·재열기와 Word/LibreOffice 왕복이 함께 검증돼야 한다.
3. 한 버전에는 하나의 주제만 넣는다.
4. 표·이미지처럼 문서 모델을 크게 바꾸는 기능은 별도 버전으로 분리한다.
5. 설치 프로그램은 기능 버전마다 다시 설계하지 않고 기존 배포 체계를 재사용한다.
6. 외부 문서 처리 라이브러리는 자체 구현 비용과 호환성 위험이 커지는 시점에 별도 승인 후 검토한다.

## 지금 개발하지 않는 기능

- 공동 편집과 클라우드 저장
- 변경 내용 추적과 주석
- 매크로와 플러그인
- 암호화 문서 편집
- 고급 스타일과 테마
- 맞춤법·문법 검사

이 기능들은 현재 제품 구조와 규모에 비해 복잡도가 높으므로 v2.0.0 이후 재검토한다.

## 로드맵 변경 절차

1. 기획자가 변경 이유와 영향을 기록한다.
2. 사용자 승인을 받는다.
3. 버전 이력과 해당 버전 기획서를 함께 갱신한다.
4. 개발자·테스터·Git 관리자 인수인계 메시지를 새 버전 폴더에 만든다.
