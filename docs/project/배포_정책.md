# 간단 워드 편집기 배포 정책

## 결정 사항

일반 사용자용 최종 배포는 ZIP이 아니라 EXE 설치 프로그램을 사용한다. 사용자는 GitHub Releases에서 설치 EXE 하나를 내려받고 한 번 설치한다. 설치 후에는 시작 메뉴나 바탕 화면의 `SimpleWordEditor.exe`를 실행한다.

별도 서버는 운영하지 않는다. GitHub Releases가 설치 파일과 버전별 배포물을 제공한다.

## 지금 설치 파일을 만들지 않는 이유

현재 프로그램은 개발 중이며 자동 테스트 1건이 실패하고 실제 Word/LibreOffice 호환성도 검증되지 않았다. 지금 설치 프로그램을 만들면 개발자가 변경할 때마다 설치와 제거를 반복해야 하고, 검증되지 않은 배포물이 누적된다.

따라서 개발 중에는 다음 방식만 사용한다.

- 개발자: Debug 또는 Release 빌드를 프로젝트에서 직접 실행
- 테스터: 전달받은 빌드 폴더 또는 같은 커밋에서 빌드해 실행
- Git 관리자: 소스·테스트·문서만 Git으로 관리
- 일반 사용자용 설치 파일: 배포 승인 직전에만 생성

## 최종 배포 형태

```text
GitHub Release v1.0.0
├─ SimpleWordEditor-Setup-v1.0.0.exe
├─ SimpleWordEditor-Setup-v1.0.0.exe.sha256
└─ 릴리스 설명
```

설치 프로그램 내부에는 Windows x64용 자체 포함 WPF 앱과 필요한 .NET/WPF 파일을 넣는다. 사용자가 .NET이나 DLL을 별도로 찾아 설치하지 않게 한다.

## Git 관리 원칙

- EXE, DLL, 설치 파일, ZIP, `bin/`, `obj/`를 일반 Git 이력에 넣지 않는다.
- 설치 EXE와 체크섬은 GitHub Release 자산으로만 올린다.
- GitHub가 자동 생성하는 소스 ZIP은 설치 파일이 아니라고 명시한다.
- 정식 태그와 Release는 기획자의 배포 승인 후 Git 관리자가 생성한다.
- 배포 파일 이름에는 제품명, 버전, 대상 운영체제를 포함한다.

권장 파일명:

```text
SimpleWordEditor-Setup-v1.0.0-win-x64.exe
SimpleWordEditor-Setup-v1.0.0-win-x64.exe.sha256
```

## 설치 프로그램 제작 시작 조건

다음 조건을 모두 충족해야 V100-P10을 시작한다.

- v1.0.0 자동 회귀 테스트 전체 통과
- 수동 UI 핵심 흐름 통과
- Microsoft Word 및 LibreOffice DOCX 왕복 통과
- DOC 지원을 유지한다면 Word COM DOC 왕복 통과
- P0/P1 결함 0건
- 기획자의 배포 후보 승인

## 설치 프로그램 완료 조건

- Windows 10/11 x64에서 설치 성공
- 관리자 권한 필요 여부가 문서와 일치
- 시작 메뉴 및 선택적 바탕 화면 바로가기 생성
- 설치 후 `SimpleWordEditor.exe` 정상 실행
- 프로그램 추가/제거에 등록
- 제거 후 사용자 문서를 삭제하지 않음
- 재설치와 버전 업그레이드 성공
- 개발 도구와 .NET이 없는 PC에서 실행 성공
- 설치 EXE SHA-256 제공
- 코드 서명 미적용 시 SmartScreen 경고 가능성을 릴리스 설명에 명시

## 향후 업데이트

v1.0.0은 GitHub Releases를 통한 수동 업데이트로 시작한다. 자동 업데이트는 에디터와 설치 체계가 안정화된 후 별도 기능 버전으로 기획한다.
