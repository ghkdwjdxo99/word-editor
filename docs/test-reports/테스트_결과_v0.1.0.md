## 4. 결함 상세

### BUG-001 — 새 문서의 초기 제목에 `*`가 표시되지 않음

- 심각도: 낮음
- 재현 절차:
  1. 프로그램을 실행한다.
  2. 초기 창 제목을 확인한다.
- 기대 결과: 기획서 4.4에 따라 `제목 없음* - 간단 워드 편집기`가 표시된다.
- 실제 결과: `제목 없음 - 간단 워드 편집기`가 표시된다.
- 원인: `NewDocument(force: true)`가 새 문서의 `state.IsDirty`를 `false`로 설정하고, 제목의 별표는 `IsDirty`가 참일 때만 추가한다.
- 영향: 새 문서가 아직 저장되지 않았다는 표시가 기획서와 다르다. 편집을 시작하면 별표는 표시된다.
- 관련 파일: `SimpleWordEditor/MainWindow.xaml.cs`

### BUG-002 — 중첩 인라인의 텍스트가 저장 시 누락될 수 있음

- 심각도: 높음 (데이터 유실 가능)
- 발생 조건: RichTextBox 문단 안에 `Span`, `Bold`, `Italic`, `Underline`, `Hyperlink` 같은 컨테이너 인라인이 있고, 실제 `Run`이 그 하위에 위치한 상태에서 DOCX로 저장한다. 서식 있는 콘텐츠를 붙여넣을 때 이런 구조가 만들어질 수 있다.
- 기대 결과: 지원 범위의 본문 텍스트와 굵게/기울임/밑줄 서식은 DOCX에 저장된다. 지원하지 않는 컨테이너가 있더라도 본문 텍스트는 보존되어야 한다.
- 실제 결과: `RichTextConverter.FromEditor`는 문단의 최상위 `Inline`만 순회하며, 최상위 항목이 `Run` 또는 `LineBreak`가 아니면 건너뛴다. 따라서 컨테이너 아래의 텍스트 전체가 문서 모델에 들어가지 않을 수 있다.
- 재현 예:
  1. 서식 있는 텍스트를 다른 프로그램에서 복사하여 붙여넣거나, 중첩 인라인 구조를 가진 FlowDocument를 연다.
  2. DOCX로 저장한다.
  3. 저장 파일을 다시 연다.
  4. 중첩 컨테이너 아래에 있던 텍스트가 누락되었는지 확인한다.
- 권장 수정: 문단의 인라인 트리를 재귀 순회하고 상위 서식을 상속하여 모든 `Run`과 `LineBreak`를 모델로 변환한다. 저장 전 붙여넣기 시나리오를 자동 회귀 테스트로 추가한다.
- 관련 파일: `SimpleWordEditor/RichTextConverter.cs`

## 6. 잔여 위험 및 권장 후속 테스트

1. Microsoft Word가 설치된 Windows PC에서 DOC 열기·수정·저장과 COM 프로세스/임시 파일 정리를 확인한다.
2. 생성 DOCX를 Microsoft Word와 LibreOffice Writer에서 열어 실제 호환성을 검증한다.
3. 대화상자를 포함한 새 문서·열기·종료의 Yes/No/Cancel 흐름을 UI 자동화 또는 수동 테스트로 확인한다.
4. 서식 있는 붙여넣기, 여러 Run이 섞인 선택, 중첩 Span을 대상으로 BUG-002 회귀 테스트를 추가한다.
5. 읽기 전용 파일, 다른 프로세스가 점유한 파일, 디스크 공간 부족을 실제 환경에서 확인한다.

## 7. 판정

조건부 통과. 핵심 DOCX 서비스와 안전 저장은 검증 범위에서 정상이나, 본문 유실 가능성이 있는 BUG-002를 수정하고 외부 호환성 및 DOC 연동을 별도 환경에서 확인하기 전에는 최종 배포 판정을 권장하지 않는다.
