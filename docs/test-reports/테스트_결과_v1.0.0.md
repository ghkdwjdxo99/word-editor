- 화면에 표시되는 중첩 인라인 본문과 지원 서식은 모델 변환에서 더 이상 조용히 누락되지 않는다.
- 표·목록·Section은 현재 DOCX 모델이 표현할 수 없으므로 저장 전에 차단한다. 다만 사용자 메시지가 차단 원인을 설명하지 못한다.
- 비지원 블록이 있을 때 파일 쓰기와 안전 저장을 시작하지 않으므로 기존 원본은 변경되지 않는다.

## 확인된 결함

### BUG-V100-001 — 비지원 편집 블록 저장 차단 시 잘못된 사용자 안내

- 심각도: 보통
- 관련 요구사항: V100-P02, V100-P09
- 발생 조건: 편집기에 `Table`, `List` 또는 `Section`이 포함된 상태에서 저장한다.
- 기대 결과: 저장을 중단하고 비지원 블록 종류와 일반 텍스트로 다시 붙여넣는 등의 해결 방법을 안내한다.
- 실제 결과: 저장은 안전하게 중단되지만 `UnsupportedEditorContentException`이 `UserErrorClassifier`에서 분류되지 않아 `Unknown`으로 처리된다. 사용자에게는 `다시 시도하거나 다른 파일을 선택해 주세요.`가 표시된다.
- 재현 결과: `Kind=Unknown`, `Action=다시 시도하거나 다른 파일을 선택해 주세요.`
- 영향: 데이터나 기존 파일은 손상되지 않지만, 사용자는 저장 실패 원인과 해결 방법을 알 수 없다.
- 원인: `UserErrorClassifier.Classify`에 `UnsupportedEditorContentException` 분기가 없다.
- 권장 수정: 해당 예외를 `UserErrorKind.Unsupported`로 분류하고 `BlockTypes`를 포함한 해결 안내를 반환한다.
- 회귀 테스트: `V100-P02 비지원 편집 블록 사용자 안내`를 추가했으며 현재 실패한다.

## 남은 위험

1. 브라우저, Word 등 외부 프로그램별 실제 클립보드 데이터 형식은 수동 붙여넣기 시험이 더 필요하다.
2. 비지원 블록은 텍스트 평탄화가 아니라 저장 차단 방식이므로 사용자가 일반 텍스트로 다시 붙여넣어야 한다.
3. Microsoft Word와 LibreOffice의 실제 DOCX 호환성은 V100-P04에서 검증해야 한다.
4. DOC COM 왕복과 `WINWORD.EXE`/임시 파일 정리는 V100-P05에서 검증해야 한다.
5. 실제 대화상자와 도구 모음을 마우스·키보드로 조작하는 수동 UI 검증은 테스터 단계에서 추가로 필요하다.
6. 현재 환경에는 Microsoft Word와 LibreOffice가 모두 설치되어 있지 않아 P04/P05는 실행할 수 없었다.
