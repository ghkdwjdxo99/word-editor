# 역할별 작업 메시지 관리 방법

이 폴더는 기획자가 개발자, 테스터, Git 관리자에게 전달할 작업 메시지를 사람이 읽기 쉬운 형태로 관리한다.

## 폴더 규칙

```text
handoffs/
└─ v1.0.0/
   ├─ README.md
   ├─ 개발자_작업요청.md
   ├─ 테스터_작업요청.md
   └─ Git관리자_작업요청.md
```

버전마다 폴더를 하나 만들고 역할별 메시지는 한 파일씩만 유지한다. 같은 역할의 메시지를 여러 파일로 복제하지 않는다.

## 상태 표시

각 메시지 상단에 다음 중 하나를 표시한다.

- 대기: 선행 작업이 끝나지 않아 시작하면 안 됨
- 진행 가능: 담당자가 바로 시작할 수 있음
- 진행 중: 담당자가 작업하고 있음
- 검토 요청: 다음 역할의 확인이 필요함
- 완료: 완료 조건과 증거가 확인됨
- 보류: 결함이나 환경 문제로 중단됨

## 갱신 책임

- 기획자: 목표, 우선순위, 완료 조건, 상태 지정
- 개발자: 구현 결과와 변경 파일 기록
- 테스터: 검증 결과와 결함 ID 기록
- Git 관리자: 커밋·태그·Release 상태 기록

담당자는 기존 지시를 삭제하지 않고 파일 아래의 “작업 결과”에 결과를 추가한다. 요구사항이 바뀌면 기획자가 변경 사유와 날짜를 기록한다.

## 읽는 순서

1. 해당 버전 폴더의 `README.md`
2. 자신의 역할별 작업 요청
3. 연결된 개발 기획서
4. 이전 역할의 작업 결과와 테스트 보고서
