# v1.1.0 개발자 작업 요청

상태: 진행 가능  
대상 기능: V110-F01 찾기  
우선순위: P1

## 전달 메시지

간단 워드 편집기 v1.1.0의 첫 작업으로 V110-F01 찾기 기능만 구현해 주세요. 바꾸기, 글꼴, 글자색과 설치 프로그램은 아직 구현하지 마세요.

## 요구사항

1. Ctrl+F로 찾기 UI를 열 수 있어야 합니다.
2. 현재 문서의 표시 본문에서 한글·영문·숫자·공백을 검색해야 합니다.
3. 다음 결과와 이전 결과로 이동할 수 있어야 합니다.
4. 대소문자 구분 옵션을 제공해야 합니다.
5. 결과가 없으면 명확하게 안내해야 합니다.
6. 검색 UI를 닫아도 문서 본문과 저장 상태가 변경되면 안 됩니다.
7. 검색 이동은 문서를 수정한 것으로 처리하지 않습니다.
8. 기존 단축키와 충돌하면 안 됩니다.

## 설계 원칙

- 검색 로직과 WPF UI를 분리해 자동 테스트할 수 있게 합니다.
- 검색 문자열이 비어 있으면 검색하지 않습니다.
- 문단 경계 처리 정책을 코드와 테스트에서 일치시킵니다.
- 첫 구현에서는 정규식, 여러 문서 검색, 검색 결과 전체 강조를 넣지 않습니다.
- 외부 패키지를 추가하지 않습니다.

## 필수 자동 테스트

- 한글·영문·숫자 검색
- 결과 없음
- 빈 검색어
- 대소문자 구분 켜기/끄기
- 여러 결과의 다음/이전 이동
- 문서 처음과 끝에서의 순환 정책
- 검색 후 본문과 `IsDirty` 미변경
- 기존 v1.0.0 회귀 테스트 전체 통과

### 2026-07-12 개발자 결과

- 상태: 개발 완료, 독립 테스터 검증 대기
- 구현: Ctrl+F 찾기 패널, 다음/이전 순환, 대소문자 구분, 결과 없음/빈 검색어 안내
- 분리 설계: `FindEngine`은 문자열 검색과 탐색 순환, `FlowDocumentSearchSnapshot`은 WPF 본문·선택 위치 매핑 담당

- 상태 보호: 검색은 선택만 변경하고 본문 및 `DocumentState.IsDirty`를 변경하지 않으며 패널 종료 시 기존 선택 복원
- 자동 검증: Debug/Release 빌드 경고 0·오류 0, 콘솔 회귀 51건 통과·0건 실패
- UI 하네스: Ctrl+F 패널, 2개 결과 이동, 패널 종료, 본문·변경 상태 보존 확인
- 제외 준수: 바꾸기, 글꼴, 글자색, 설치 프로그램 미구현
- 남은 위험: 현재 검색 스냅샷은 v1.0.0의 지원 본문인 최상위 Paragraph와 중첩 Inline을 대상으로 하며 저장 차단 대상인 Table/List/Section 내부는 검색하지 않음
