# v1.0.0 테스터 작업 요청

상태: 진행 가능 — 현재 자동 테스트 38건 통과 상태  
검증 대상: V100-P11 및 v1.0.0 핵심 UI

## 테스터에게 전달할 메시지

개발자가 V100-P11 완료를 보고한 뒤 독립 검증해 주세요. 개발자의 테스트 결과를 그대로 승인 근거로 사용하지 말고 동일 커밋에서 다시 검증해야 합니다.

## 자동 검증

1. Debug 및 Release 빌드가 경고·오류 없이 성공하는지 확인합니다.
2. 전체 자동 회귀 테스트를 실행합니다.
3. 기존 37건과 V100-P11 테스트가 모두 통과하는지 확인합니다.
4. 테스트 삭제, 건너뛰기, 기대값 완화 여부를 확인합니다.

## 수동 UI 검증

- Table 저장 차단 메시지에 원인과 해결 방법이 표시됨
- List 저장 차단 메시지에 원인과 해결 방법이 표시됨
- Section 저장 차단 메시지에 원인과 해결 방법이 표시됨
- 저장 실패 후 원본 파일이 유지됨
- 새 문서·열기·저장·다른 이름으로 저장 동작
- 변경 후 새 문서·열기·종료에서 저장/폐기/취소 동작
- 굵게·기울임·밑줄·크기·정렬의 단일 및 혼합 선택 표시
- 확장자가 없거나 잘못된 저장 이름 처리

## 판정 규칙

- 데이터 손실, 기존 파일 손상, 강제 종료: 즉시 실패
- 원인이나 해결 방법이 없는 저장 차단 메시지: 실패
- 자동 테스트 1건이라도 실패: 실패
- 기능은 동작하지만 문구만 다소 어색함: 결함으로 기록 후 심각도에 따라 조건부 판정
