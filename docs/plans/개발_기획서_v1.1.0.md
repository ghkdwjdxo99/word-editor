# 간단 워드 편집기 v1.1.0 개발 기획서

> 문서 버전: v1.1.0  
> 상태: 개발 착수 승인 — V110-F01부터 진행  
> 착수 조건: v1.0.0 자동·수동 UI 테스트 통과, Word 검증은 배포 전까지 보류  
> 선행 문서: `docs/plans/개발_기획서_v1.0.0.md`

## 1. 목표

v1.1.0은 기본 문서 작성 생산성을 높이는 첫 기능 버전이다. 찾기/바꾸기와 글꼴·글자색을 추가한다. 이미지, 표, 목록, 인쇄는 포함하지 않는다.

## 2. 추가 기능

### V110-F01 — 찾기

- Ctrl+F로 찾기 창 또는 패널 열기
- 현재 문서 본문에서 문자열 검색
- 다음/이전 결과 이동
- 대소문자 구분 선택
- 검색 결과 없음 안내
- 검색 UI를 닫아도 문서 선택과 본문이 손상되지 않음

### V110-F02 — 바꾸기

- Ctrl+H로 바꾸기 UI 열기
- 현재 결과 한 건 바꾸기
- 전체 바꾸기
- 전체 바꾸기 전 대상 건수 표시 또는 실행 후 변경 건수 안내
- 실행 취소 한 번 이상으로 변경을 되돌릴 수 있어야 함
- 빈 검색어로 전체 바꾸기 금지

### V110-F03 — 글꼴

- Windows에 설치된 글꼴 목록 제공
- 선택 영역에 글꼴 적용
- 커서 위치와 혼합 선택 상태 표시
- 기본 글꼴은 맑은 고딕 유지
- DOCX의 `w:rFonts`를 읽고 저장
- 없는 글꼴은 기본 글꼴로 대체하되 본문은 보존

### V110-F04 — 글자색

- 기본 색상 목록 제공
- 선택 영역에 글자색 적용
- 혼합 선택 상태 표시
- DOCX의 `w:color`를 읽고 저장
- 잘못된 색상 값은 검정으로 대체하고 본문은 보존

## 3. 내부 모델 변경

`TextRun`에 글꼴 이름과 RGB 글자색을 추가한다. 기존 v1.0.0 문서를 열 때 값이 없으면 기본 글꼴과 검정색을 사용한다.

모델 변경 후 다음 항목을 반드시 검증한다.

- 기존 굵게·기울임·밑줄·크기 유지
- 글꼴·색상별 Run 분리와 병합
- 한글과 영문이 섞인 글꼴 처리
- 기존 DOCX를 열고 저장할 때 본문 손실 없음

## 4. 제외 범위

- 글자 배경색
- 위첨자·아래첨자
- 문자 간격
- 스타일 및 테마 글꼴
- 정규식 검색
- 여러 파일 검색
- 이미지·표·목록·인쇄·PDF

## 5. 개발 순서

1. V110-F01 찾기
2. V110-F02 바꾸기
3. V110-F03 내부 모델 및 DOCX 글꼴
4. V110-F04 내부 모델 및 DOCX 글자색
5. 도구 모음 상태 동기화
6. 전체 자동 회귀 테스트
7. Word/LibreOffice 왕복 검증

한 기능씩 구현하고 해당 자동 테스트가 통과한 뒤 다음 기능으로 이동한다.

## 6. 필수 테스트

- 한글·영문·숫자 찾기
- 다음/이전 순환과 결과 없음
- 대소문자 구분
- 한 건 및 전체 바꾸기
- 빈 검색어 차단
- 바꾸기 실행 취소
- 단일·혼합 글꼴 선택
- 단일·혼합 색상 선택
- 글꼴·색상 DOCX 저장 및 재열기
- 기존 v1.0.0 테스트 전체 통과
- Word와 LibreOffice에서 글꼴·색상 확인
- Word/LibreOffice 저장본을 편집기에서 재열기

## 7. 완료 조건

- V110-F01~F04 구현 및 자동 테스트 통과
- v1.0.0 회귀 결함 0건
- 화면 본문과 저장 본문 불일치 0건
- Word/LibreOffice 왕복 통과
- P0/P1 결함 0건
- 테스터 독립 승인

## 8. 착수 및 보류 조건

사용자 결정에 따라 Word를 구할 수 없는 현재 환경에서는 V100-P04~P05를 배포 전 보류 항목으로 남기고 v1.1.0 개발을 시작한다.

다음 중 하나라도 해당하면 개발자는 다음 기능으로 넘어가지 않는다.

- 현재 기능의 자동 테스트 실패
- 데이터 유실 또는 원본 손상 결함 존재
- 테스터가 P0/P1 결함을 등록함
- 기획되지 않은 모델 변경이 필요함

Word/LibreOffice와 DOC 호환성 검증은 설치 프로그램 제작 및 정식 배포 전에 반드시 다시 확인한다. Word를 끝내 구할 수 없다면 DOC 기능을 정식 지원으로 표시하지 않고 시험적 기능으로 남기거나 배포판에서 숨기는 별도 결정을 내려야 한다.

## 9. 개발자에게 전달할 예정 메시지

`docs/workflow/handoffs/v1.1.0/개발자_작업요청.md`를 기준으로 V110-F01 찾기만 먼저 구현한다. 찾기 기능이 개발자 자체 테스트와 테스터 독립 검증을 통과한 뒤 V110-F02 바꾸기를 요청한다.
