# 초간단 DOC/DOCX 워드 편집기 v0.1.0 개발 기획서

> 문서 버전: v0.1.0  
> 목표 버전: v0.1.0 시제품  
> 상태: 개발 완료  
> 후속 기획: `docs/plans/개발_기획서_v1.0.0.md`

## 1. 문서 목적

이 문서는 Windows에서 실행되는 아주 단순한 문서 편집기의 1차 버전(MVP)을 개발하기 위한 구현 기준서다.

프로그램의 목적은 다음 세 가지다.

- 새 문서를 만든다.
- 기존 `.docx` 또는 `.doc` 문서를 열어 내용을 편집한다.
- 편집 결과를 저장하거나 다른 이름으로 저장한다.

표, 이미지, 각주, 변경 내용 추적, 공동 편집 같은 고급 기능은 모두 제외한다. 사용자가 문서의 본문 텍스트를 작성하고 최소한의 서식을 적용하는 것만 지원한다.

## 2. 가장 중요한 기술 결정

### 2.1 개발 대상

- 운영체제: Windows 10/11
- 애플리케이션 형태: 단일 창 Windows 데스크톱 프로그램
- 권장 기술: C# + .NET + WPF
- 편집 화면: WPF `RichTextBox`
- 파일 선택 창: Windows 기본 열기/저장 대화상자
- 기본 저장 형식: `.docx`

WPF와 .NET 기본 API만 사용하면 별도 UI 프레임워크나 문서 처리 패키지를 추가하지 않고 개발할 수 있다. 개발 PC에 Visual Studio 또는 .NET SDK가 없다면 개발 도구 설치는 필요하지만, 실행 프로그램에는 별도 문서 라이브러리를 포함하지 않는 것을 원칙으로 한다.

### 2.2 DOCX와 DOC의 차이 및 지원 원칙

`.docx`는 ZIP 안에 XML 파일이 들어 있는 공개 규격이므로 .NET의 압축 및 XML API로 최소 기능을 직접 구현한다.

`.doc`는 구형 바이너리 규격이며 구조가 매우 복잡하다. 외부 라이브러리도 사용하지 않고 Microsoft Word도 없는 환경에서 `.doc`를 직접 생성·편집하는 기능은 MVP 범위에서 현실적으로 구현할 수 없다. 파일 확장자만 `.doc`로 바꾼 RTF나 HTML을 저장하는 방식은 진짜 DOC 파일이 아니므로 사용하지 않는다.

따라서 아래와 같이 처리한다.

| 형식 | Microsoft Word 미설치 PC | Microsoft Word 설치 PC |
|---|---|---|
| DOCX 새로 만들기 | 지원 | 지원 |
| DOCX 열기/편집/저장 | 지원 | 지원 |
| DOC 새로 만들기 | 지원하지 않음 | Word 자동화로 지원 |
| DOC 열기/편집/저장 | 지원하지 않음 | Word 자동화로 지원 |
| DOC를 DOCX로 변환 | 지원하지 않음 | Word 자동화로 지원 |

Word 자동화는 사용자의 PC에 이미 설치된 Microsoft Word의 COM 기능을 호출한다. Word가 없으면 DOC 메뉴를 비활성화하거나, DOC 파일 선택 시 “DOC 파일을 사용하려면 Microsoft Word가 설치되어 있어야 합니다. DOCX 파일은 Word 없이 사용할 수 있습니다.”라고 안내한다.

독립 실행형 DOC 지원이 반드시 필요해지는 시점에는 LibreOffice 또는 검증된 상용/오픈소스 문서 라이브러리 도입을 별도 프로젝트로 검토한다. 이는 MVP에 포함하지 않는다.

## 3. MVP 기능 범위

### 3.1 반드시 구현할 기능

1. 새 문서
   - 빈 편집 화면을 만든다.
   - 저장 전 임시 문서 상태로 표시한다.

2. 파일 열기
   - `.docx`를 연다.
   - Word 설치 환경에서는 `.doc`도 연다.
   - 편집기가 지원하지 않는 요소는 가능한 범위에서 본문 텍스트만 읽는다.

3. 본문 편집
   - 텍스트 입력 및 삭제
   - 줄바꿈과 문단 구분
   - 잘라내기, 복사, 붙여넣기
   - 실행 취소, 다시 실행

4. 최소 서식
   - 굵게
   - 기울임
   - 밑줄
   - 글자 크기: 8~72pt
   - 문단 정렬: 왼쪽, 가운데, 오른쪽

5. 저장
   - 현재 파일에 저장
   - 다른 이름으로 저장
   - 기본 형식은 `.docx`
   - Word 설치 환경에서는 `.doc` 저장 선택 제공

6. 안전한 종료
   - 문서가 변경된 뒤 닫기, 새 문서, 다른 파일 열기, 프로그램 종료를 시도하면 저장 여부를 묻는다.
   - 선택지는 저장, 저장하지 않음, 취소다.

### 3.2 구현하지 않을 기능

- 표, 이미지, 도형, 차트, 수식
- 머리글/바닥글, 페이지 번호
- 각주, 미주, 목차
- 목록 번호 및 다단계 목록
- 스타일 관리, 테마, 글꼴 포함
- 맞춤법 검사, 자동 교정
- 페이지 레이아웃, 용지 크기, 여백, 인쇄, PDF 변환
- 암호화 문서, 전자 서명
- 변경 내용 추적, 주석, 공동 편집
- 매크로, `.docm`, `.dot`, `.dotx`
- 클라우드 저장 및 자동 저장

지원하지 않는 요소가 포함된 문서를 열었다가 저장하면 해당 요소가 사라질 수 있다. 이를 숨기지 말고 최초 저장 전에 경고해야 한다.

## 4. 화면 구성

한 개의 창만 사용한다.

### 4.1 상단 메뉴

- 파일
  - 새 문서
  - 열기
  - 저장
  - 다른 이름으로 저장
  - 종료
- 편집
  - 실행 취소
  - 다시 실행
  - 잘라내기
  - 복사
  - 붙여넣기
  - 전체 선택

### 4.2 도구 모음

- 굵게, 기울임, 밑줄 버튼
- 글자 크기 선택 상자
- 왼쪽, 가운데, 오른쪽 정렬 버튼

### 4.3 본문 영역

- 창의 남은 공간을 채우는 `RichTextBox`
- 세로 스크롤 제공
- 기본 글자 크기 11pt
- 기본 문단 정렬은 왼쪽
- 페이지 단위 화면은 구현하지 않고 연속 편집 화면으로 표시

### 4.4 상태 및 제목 표시

- 창 제목 형식: `파일명 - 간단 워드 편집기`
- 저장되지 않은 변경 사항이 있으면 파일명 뒤에 `*` 표시
- 새 문서는 `제목 없음* - 간단 워드 편집기`로 표시
- 상태 표시줄에는 현재 파일 형식과 Word 연동 가능 여부만 표시

## 5. 사용자 흐름

### 5.1 새 문서 작성

1. 프로그램을 실행한다.
2. 빈 문서가 나타난다.
3. 사용자가 내용을 입력하고 서식을 적용한다.
4. 저장을 선택한다.
5. 저장 위치와 파일명을 선택한다.
6. `.docx`로 저장한다.

### 5.2 기존 DOCX 편집

1. 열기를 선택한다.
2. `.docx` 파일을 고른다.
3. 프로그램이 지원 가능한 본문과 서식을 읽어 화면에 표시한다.
4. 지원하지 않는 요소가 발견되면 상단 또는 대화상자에 경고한다.
5. 사용자가 수정한 뒤 저장한다.
6. 원본 교체는 임시 파일 저장이 성공한 후에만 수행한다.

### 5.3 기존 DOC 편집

1. 프로그램 시작 시 Microsoft Word 연동 가능 여부를 확인한다.
2. Word가 있으면 DOC 파일을 Word 자동화를 통해 임시 DOCX로 변환한다.
3. 임시 DOCX를 편집기에 연다.
4. 저장할 때 다시 DOC 형식으로 내보낸다.
5. 변환과 저장 작업이 끝나면 Word 프로세스와 임시 파일을 정리한다.
6. Word가 없으면 파일을 열지 않고 안내 메시지를 표시한다.

## 6. 프로그램 내부 구조

UI와 문서 처리를 분리한다. 최소한 다음 책임 단위로 구성한다.

| 구성 요소 | 책임 |
|---|---|
| 메인 창 | 메뉴, 도구 모음, 편집 화면, 사용자 명령 연결 |
| 문서 상태 관리자 | 현재 경로, 파일 형식, 변경 여부, 새 문서 여부 관리 |
| 편집 모델 변환기 | RichTextBox의 문단/텍스트/서식을 내부 문서 모델로 변환 |
| DOCX 판독기 | DOCX ZIP과 XML을 읽어 내부 문서 모델 생성 |
| DOCX 기록기 | 내부 문서 모델로 최소 DOCX 패키지 생성 |
| Word 연동기 | Word 설치 확인, DOC↔DOCX 변환, Word 프로세스 정리 |
| 파일 안전 저장기 | 임시 파일 작성, 검증, 원본 교체, 실패 시 복구 |
| 경고/오류 처리기 | 지원하지 않는 요소와 파일 손상, 권한 오류 안내 |

내부 문서 모델은 문서 → 문단 → 텍스트 구간의 단순 구조로 제한한다. 텍스트 구간에는 문자열, 굵게, 기울임, 밑줄, 글자 크기만 둔다. 문단에는 정렬 값만 둔다. DOCX의 복잡한 구조를 그대로 편집 화면에 보존하려고 하지 않는다.

## 7. DOCX 최소 구현 규격

### 7.1 사용할 기본 API

- 압축 읽기/쓰기: `System.IO.Compression`
- XML 읽기/쓰기: `System.Xml`
- 파일 처리: `System.IO`

Open XML SDK 같은 NuGet 패키지는 사용하지 않는다.

### 7.2 새 DOCX에 필요한 최소 패키지 항목

새 문서를 저장할 때 최소한 다음 항목을 생성한다.

- `[Content_Types].xml`
- `_rels/.rels`
- `word/document.xml`
- `word/_rels/document.xml.rels`

필요한 경우 기본 문서 속성 파일을 추가할 수 있지만 MVP 필수 사항은 아니다. 생성 파일은 Microsoft Word와 LibreOffice에서 열리는지 검증한다.

### 7.3 읽을 DOCX 요소

- 본문: `w:document`, `w:body`
- 문단: `w:p`
- 텍스트 구간: `w:r`
- 텍스트: `w:t`
- 줄바꿈 및 탭: `w:br`, `w:tab`
- 텍스트 서식: `w:b`, `w:i`, `w:u`, `w:sz`
- 문단 정렬: `w:jc`

XML 네임스페이스를 접두사 문자열로 비교하지 말고 네임스페이스 URI와 로컬 이름으로 판별한다. 텍스트 앞뒤 공백 보존을 위해 `xml:space="preserve"`를 처리한다.

### 7.4 저장할 DOCX 요소

- 문서와 본문
- 문단
- 텍스트 구간
- 굵게, 기울임, 밑줄, 글자 크기
- 왼쪽, 가운데, 오른쪽 정렬
- 줄바꿈과 탭

DOCX의 글자 크기는 half-point 단위라는 점을 반영한다. 예를 들어 화면의 11pt는 XML에서 22로 기록한다.

### 7.5 기존 파일을 여는 정책

기존 DOCX를 완전 보존하는 편집은 목표가 아니다. 파일을 열 때 필요한 본문 내용을 내부 모델로 가져오고 저장 시 지원하는 요소만으로 새 DOCX를 다시 만든다.

다음 요소가 발견되면 `hasUnsupportedContent` 상태를 켠다.

- 표, 이미지, 도형
- 머리글/바닥글 참조
- 각주/미주
- 주석 및 변경 추적
- 하이퍼링크
- 섹션 및 복잡한 페이지 설정
- 필드, 수식, 콘텐츠 컨트롤

이 상태의 문서를 처음 저장할 때 “이 문서에는 지원하지 않는 요소가 포함되어 있습니다. 저장하면 해당 요소가 제거될 수 있습니다. 다른 이름으로 저장하는 것을 권장합니다.”라고 안내한다. 기본 선택은 다른 이름으로 저장으로 둔다.

## 8. DOC 연동 구현 기준

Word 연동기는 다른 구성 요소와 완전히 분리한다. 프로그램의 기본 DOCX 기능은 Word가 없어도 동작해야 한다.

- 프로그램 시작 시 Word COM 객체 생성 가능 여부를 짧게 확인하고 즉시 종료한다.
- DOC를 열 때 Word 창은 보이지 않게 실행한다.
- 원본 DOC를 직접 편집하지 않고 임시 폴더의 DOCX 사본으로 변환한다.
- DOC 저장 시 임시 DOCX를 Word로 열어 DOC 형식으로 저장한다.
- 모든 COM 객체는 작업 성공 여부와 관계없이 해제한다.
- Word가 이미 실행 중이어도 사용자의 기존 Word 창과 문서를 종료하지 않는다.
- 변환 실패, 암호 문서, 손상 문서는 편집하지 않고 원본을 유지한다.
- 비정상 종료 뒤 남은 임시 파일은 다음 실행 시 정리한다.

Word 자동화는 데스크톱 사용자 환경에서만 사용하며 서버 서비스처럼 운용하지 않는다.

## 9. 저장 안정성

원본 파일에 바로 쓰지 않는다.

1. 원본과 같은 폴더에 임시 파일을 만든다.
2. 임시 DOCX 패키지를 완전히 기록한다.
3. ZIP을 다시 열 수 있는지 확인한다.
4. 필수 XML 항목이 존재하고 XML 파싱이 되는지 확인한다.
5. 검증이 성공하면 원본을 교체한다.
6. 교체 실패 시 원본은 그대로 두고 임시 파일을 삭제한다.

파일이 다른 프로그램에서 사용 중이거나 쓰기 권한이 없으면 저장 실패 이유를 사용자에게 표시한다. 저장에 실패한 경우 변경 상태 `*`를 제거하지 않는다.

## 10. 오류 및 예외 처리

다음 상황을 반드시 구분해 안내한다.

- 파일이 존재하지 않음
- 파일 읽기 또는 쓰기 권한 없음
- 다른 프로그램이 파일을 사용 중임
- ZIP 또는 XML이 손상된 DOCX
- 암호화된 문서
- 지원하지 않는 파일 형식
- Word 미설치 상태에서 DOC 요청
- Word 자동화 또는 변환 실패
- 디스크 공간 부족

오류 메시지는 기술적인 예외 문자열만 보여주지 말고, 사용자가 취할 행동을 함께 안내한다. 진단용 상세 오류는 로컬 로그에 기록하되 문서 본문은 로그에 기록하지 않는다.

| 굵게 | Ctrl+B |
| 기울임 | Ctrl+I |
| 밑줄 | Ctrl+U |

## 12. 구현 순서

### 1단계: 편집 화면

- 메인 창과 RichTextBox 구성
- 메뉴, 도구 모음, 단축키 연결
- 텍스트 입력과 기본 서식 동작 구현
- 문서 변경 여부 추적

완료 조건: 앱 안에서 새 문서를 작성하고 편집할 수 있다.

### 2단계: 새 DOCX 생성 및 저장

- 내부 문서 모델 구성
- 편집 화면에서 내부 모델 추출
- 최소 DOCX ZIP/XML 생성
- 저장 및 다른 이름으로 저장 구현
- 안전 저장과 저장 실패 처리 구현

완료 조건: 작성한 문서를 DOCX로 저장한 뒤 Microsoft Word와 LibreOffice에서 열 수 있다.

### 3단계: DOCX 열기

- DOCX 패키지 검증
- 본문, 문단, 지원 서식 파싱
- 내부 모델을 RichTextBox에 표시
- 지원하지 않는 요소 감지 및 경고

완료 조건: 프로그램이 저장한 DOCX를 다시 열었을 때 내용과 지원 서식이 유지된다.

### 4단계: 문서 수명 주기

- 새 문서, 다른 파일 열기, 창 닫기 전 저장 확인
- 창 제목과 변경 표시
- 최근 경로 기억 여부는 선택 사항

완료 조건: 사용자가 실수로 변경 내용을 잃는 기본 경로가 없다.

### 5단계: 선택적 DOC 연동

- Word 설치 여부 확인
- DOC를 임시 DOCX로 변환해 열기
- DOC로 다시 저장
- COM 및 임시 파일 정리

완료 조건: Word 설치 PC에서 간단한 DOC 파일의 열기, 수정, 저장이 가능하고 Word 미설치 PC에서는 명확한 안내가 나온다.

### 6단계: 배포 점검

- Release 빌드 생성
- 대상 PC에 맞춘 .NET 배포 방식 결정
- 설치 관리자 없이 폴더 복사 실행이 가능한지 확인
- 실행 파일과 필요한 런타임 파일만 배포

## 13. 테스트 기준

### 13.1 필수 기능 테스트

- 한글, 영문, 숫자, 특수문자를 입력하고 저장/재열기
- 빈 문서 저장/재열기
- 여러 문단과 빈 줄 저장/재열기
- 굵게, 기울임, 밑줄, 글자 크기 혼합 구간 저장/재열기
- 왼쪽, 가운데, 오른쪽 문단 정렬 저장/재열기
- 탭과 줄바꿈 저장/재열기
- 실행 취소/다시 실행
- 저장하지 않은 상태에서 새 문서, 열기, 종료 시 확인 창
- 읽기 전용 폴더와 사용 중인 파일에 저장 시 오류 처리
- 손상된 DOCX를 열 때 프로그램이 종료되지 않는지 확인

### 13.2 호환성 테스트

- 프로그램이 만든 DOCX를 Microsoft Word에서 열기
- 프로그램이 만든 DOCX를 LibreOffice Writer에서 열기
- Word에서 만든 단순 DOCX를 프로그램에서 열고 다시 저장하기
- 지원하지 않는 표/이미지가 있는 DOCX를 열 때 경고 표시
- Word 설치 및 미설치 환경에서 DOC 메뉴 동작 확인

### 13.3 저장 안정성 테스트

- 저장 도중 예외를 강제로 발생시켜 원본이 유지되는지 확인
- 디스크 공간 부족 상황에서 변경 상태가 유지되는지 확인
- 같은 파일을 다른 프로그램이 점유한 상태에서 원본 손상이 없는지 확인
- 비정상 종료 후 임시 파일 정리 확인

## 14. 완료 정의

다음 조건을 모두 만족하면 MVP를 완료한 것으로 본다.

- 외부 문서 처리 패키지 없이 새 DOCX를 만들 수 있다.
- 단순 DOCX의 본문과 지원 서식을 열고 수정하고 저장할 수 있다.
- 저장한 DOCX가 Microsoft Word와 LibreOffice에서 정상적으로 열린다.
- 저장하지 않은 변경 사항을 잃지 않도록 확인 절차가 동작한다.
- 잘못된 파일이나 저장 실패 때문에 프로그램이 강제 종료되지 않는다.
- 지원하지 않는 문서 요소가 제거될 수 있음을 저장 전에 알린다.
- Word 설치 PC에서는 DOC의 열기와 저장이 가능하다.
- Word 미설치 PC에서는 DOC가 지원되지 않는 이유와 DOCX 대안을 명확히 안내한다.
